Arthas的简单使用

一、Arthas介绍

官网：https://arthas.aliyun.com/doc/

这里简单来说arthas是一个单独的监控诊断程序，需要挂载到我们需要监控的Java程序。可以通过全局视角实时查看应用 load、内存、gc、线程的状态信息，并能在不修改应用代码的情况下，对业务问题进行诊断，包括查看方法调用的出入参、异常，监测方法执行耗时，类加载信息等，大大提升线上问题排查效率。

二、Arthas简单使用

1、下载与启动

curl -O https://arthas.aliyun.com/arthas-boot.jar
java -jar arthas-boot.jar

这里启动到一半时，arthas会自动帮你检索到当前运行的Java程序，并且给出它定义的序号[1]、[2]等等，我们需要选择要挂载的Java程序需要在命令行输入即可。例如：2，看到如下图代表启动成功

2、常用命令

dashboard-控制台

输入[dashboard]，按回车/enter，会展示当前Java进程的信息，按ctrl+c可以中断执行。

dashboard命令可以查看CPU、线程状态，内存信息、GC情况和jdk版本等信息

dashboard常用字段说明：


ID:Java级别的线程ID
NAME:线程名字
GROUP:线程组名
PRIORITY:线程优先级
STATE:线程状态
%CPU:该线程CPU的使用率
DELTA_TIME:上次采样后线程运行增量cpu时间
%CPU计算原理：比如采样间隔2000ms，某个线程的增量cpu时间为200ms，则cpu使用率=200/2000=10%

Memory区域主要参数说明：
heap：堆内存使用情况（ps_eden_space+ps_survivor_space+ps_old_gen）
ps_eden_space：伊甸园区内存使用情况
ps_survivor_space：幸存区内存使用情况
ps_old_gen ：老年代内存使用情况
nonheap：非堆内存使用情况

GC区域说明：
gc.ps_scavenge.count：从应用程序启动到当前采样时间年轻代gc次数
gc.ps_scavenge.time(ms)：从应用程序启动到当前采样时间年轻代gc所用的总时间(毫秒)
gc.ps_marksweep.count：从应用程序启动到当前采样时间老年代gc次数 
gc.ps_marksweep.time(ms)：从应用程序启动到当前采样时间老年代gc所用的总时间(毫秒)

这个监控页面会实时刷新，默认每5000毫秒（5秒）刷新一次。可以通过 - i 参数指定刷新频率，-n 参数指定刷新次数。这个统计会有一定的开销，所以刷新频率不要太高，建议5秒以上，刷新次数建议10次以内

// 每10秒刷新一次，3次后停止 
dashboard -i 10000 -n 3
thread-查看线程

使用thread命令可以查看线程的状态，显示的结果实际就是dashboard结果的第一栏

thread命令可以追加参数
id：可以查看指定线程id的堆栈信息 
-n value：找出最忙的value个线程，并打印堆栈信息 例如：thread -n 3 (找出最忙的3个线程)
-b：找出当前正在阻塞其他线程的线程，可以用来查找死锁
-i value：指定采样cpu占比的时间间隔，单位：毫秒。默认为100ms。

三、WebConsole使用

WebConsole 是 Arthas 提供的 web 页面，它可以让用户在自己的电脑上，连接远程服务器的Arthas执行命令，命令的执行方式和在服务器直接操作没有任何区别。

语法：java -jar arthas-boot.jar --target-ip 192.168.1.65 --http-port 8563 3558

参数说明：
java -jar arthas.jar：启动命令 
--target-ip：指定 WebConsole 连接的 ip，也就是服务器的外网ip 
--http-port：指定 WebConsole 连接的端口，也就是服务器对外暴露的端口 
3558：要监听的 java 程序的进程 id
